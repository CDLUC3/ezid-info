Creator

The main researcher involved in producing the data, or the author of the publication, in priority order.

May be a corporate/institutional or personal name.

Name
The name of the creator
Examples: Smith,John; Miller, Elizabeth
Identifier
Uniquely identifies an individual or legal entity, according to various schemes.
The format is dependent upon scheme.
Identifier Scheme
The name of the name identifier scheme.
If identifier is used, identifier scheme is mandatory. Examples: ORCID12, ISNI13
Scheme URI
The URI ofthe name identifier scheme.
Examples: http://www.isni.org or http://www.orcid.org

Title

A name or title by which a resource is known.

Subject

Subject, keyword, classification code, or key phrase describing the resource.

Scheme
The name of the subject scheme or classification code or authority if one is used.
Scheme URI
The scheme URI of the subject identifier scheme.
Examples: http://id.loc.gov/authorities/subjects; http://dewey.info/

Contributor

The institution or person responsible for collecting, managing, distributing, or otherwise contributing to the development of the resource.

Name
The name of the contributor.
If contributor is used, then name is mandatory. Examples: Patel, Emily; Doe, John

The personal name format may be: family, given. Non‐roman names should be transliterated according to the ALA‐LC schemes.
Name Identifier
Uniquely identifies an individual or legal entity, according to various schemes.
The format is dependent upon scheme.
Name Identifier Scheme
The name of the name identifier scheme.
If name identifier is used, name identifier scheme is mandatory. Examples: ORCID15, ISNI16
Scheme URI
The URI of the name identifier scheme.
Examples: http://www.isni.org, http://www.orcid.org
Contributor Type
The type of contributor of the resource.

Date

Date relevant to the work.

YYYY,YYYY‐MM‐DD, YYYY‐MM‐DDThh:mm:ssTZD or any other format or level of granularity described in W3CDTF. Use RKMS‐ISO8601 standard for depicting date ranges. Example: 2004‐03‐02/2005‐06‐02

Date Type
The type of date.
If date is used, date type is mandatory.

Alternate Identifier

An identifier or identifiers other than the primary identifier applied to the resource being registered. This may be any alphanumeric string which is unique within its domain of issue. May be used for local identifiers. Alternate identifier should be used for another identifier of the same instance (same location, same file).

Example, A local accession number: E‐GEOD‐34814

Identifier Type
The type of the alternate identifier. If alternate identifier is used, identifier type is mandatory.

Related Identifier

Identifier of related resources. Must be globally unique.

Identifier Type
The type of the related identifier
If a related identifier is used, identifier type is mandatory.
Relation Type
Description of the relationship of the resource being registered (A) and the related resource (B).
If a related identifier is used, relation type is mandatory.

Rights

Any rights information for this resource.

Rights
Provide a rights management statement for the resource or reference a service providing such information. Include embargo information if applicable.
Use the complete title of a license and include version information if applicable. Example: Creative Commons Attribution 3.0 Germany License
URI
The URI of the license.
Example: http://creativecommons.org/licenses/by/3.0/de/deed.en

Description

All additional information that does not fit in any of the other categories. May be used for technical information.

Description
It is best practice to supply a description.
Type
The type of the description.
If description is used, type is mandatory.

Geolocation

Spatial region or named place where the data was gathered or about which the data is focused.

Repeat this property to indicate several different locations.

Point
A point location in space.
A point contains a single latitude‐longitude pair, separated by whitespace. Example: 31.233-67.302
Box
The spatial limits of a place.
A box contains two white space separated latitude‐longitude pairs, with each pair separated by whitespace. The first pair is the lower corner, the second is the upper corner. Example: 42.893‐71.032 41.090‐68.211
Place
Description of a geographic location.
