Creator

The main researcher involved in producing the data, or the author of the publication, in priority order. May be a corporate/institutional or personal name.

Name
The name of the creator (Required)
Examples: Smith, John; Miller, Elizabeth

The personal name format should be: family, given. Non-roman names should be transliterated according to the ALA-LC schemes http://www.loc.gov/catdir/cpso/roman.html.
Identifier
Uniquely identifies an individual or legal entity, according to various schemes. The format is dependent upon scheme.
Identifier Scheme
The name of the identifier scheme. If identifier is used, scheme is mandatory.
Examples: ORCID12, ISNI13
Scheme URI
The URI (location) of the identifier scheme.
Examples: http://www.isni.org or http://www.orcid.org
Affiliation
The organizational or institutional affiliation of the creator

Title

A name or title by which a resource is known. (Required)

Publisher

The name of the entity that holds, archives, publishes prints, distributes, releases, issues, or produces the resource. This property will be used to formulate the citation, so consider the prominence of the role. (Required)

Examples: World Data Center for Climate (WDCC); GeoForschungsZentrum Potsdam (GFZ); Geological Institute, University of Tokyo

Publication Year

The year when the data was or will be made publicly available. If an embargo period has been in effect, use the date when the embargo period ends. (Required)

Format: YYYY

Language

The primary language of the resource.

Examples: en, de, fr

Version

The version number of the resource.

Suggested practice: track major_version.minor_version.

Register a new identifier for a major version change. Individual stewards need to determine which are major vs. minor versions.

May be used in conjunction with properties AlternateIdentifier and RelatedIdentifier to indicate various information updates.

May be used in conjunction with Description to indicate the nature and file/record range of version.

Resource Type

The general type of a resource. For definitions of the selections, consult the full online help. It is best practice to indicate the Resource Type.

Resource Type Description

Free text field available to provide a more specific description of the resource type, as appropriate. If left incomplete, this will auto-fill to the value of the Resource Type field.

Description

Additional information to describe the resource, including the abstract. May be used for technical information.

Description
It is best practice to supply an Abstract.
Type
The type of the description.
If description is used, type is mandatory.

Subject

Subject, keyword, classification code, or key phrase describing the resource.

Scheme
The name of the subject scheme or classification code or authority if one is used.
Scheme URI
The URI (location) of the subject identifier scheme.
Examples: http://id.loc.gov/authorities/subjects; http://dewey.info/

Contributor

The institution or person responsible for collecting, managing, distributing, or otherwise contributing to the development of the resource.

Name
The name of the contributor.
If contributor is used, then name is mandatory. Examples: Patel, Emily; Doe, John

The personal name format should be: family, given. Non-roman names should be transliterated according to the ALA-LC schemes http://www.loc.gov/catdir/cpso/roman.html.
Identifier
Uniquely identifies an individual or legal entity, according to various schemes.
The format is dependent upon scheme.
Identifier Scheme
The name of the identifier scheme.
If identifier is used, scheme is mandatory. Examples: ORCID15, ISNI16
Scheme URI
The URI (location) of the name identifier scheme.
Examples: http://www.isni.org, http://www.orcid.org
Contributor Type
The type, or role, of contributor of the resource.
Affiliation
The organizational or institutional affiliation of the contributor

Format

Technical format of the resource. Use file extension or MIME type where possible.

Examples: PDF, XML, MPG, application/pdf, text/xml, video/mpeg

Date

Any date relevant to the work.

YYYY,YYYY-MM-DD, YYYY-MM-DDThh:mm:ssTZD or any other format or level of granularity described in W3CDTF. Use RKMS-ISO8601 standard for depicting date ranges. http://www.ukoln.ac.uk/metadata/dcmi/collection-RKMS-ISO8601/ Example: 2004-03-02/2005-06-02

Date Type
The type of date.
If date is used, date type is mandatory.

Alternate Identifier

An identifier or identifiers other than the primary identifier applied to the resource being registered. This may be any alphanumeric string which is unique within its domain of issue. May be used for local identifiers. Alternate identifier should be used for another identifier of the same instance (i.e., same location, same file).

Example, A local accession number: E-GEOD-34814

Identifier Type
The type of the alternate identifier. If alternate identifier is used, identifier type is mandatory.

Related Identifier

An identifier or identifiers of related resources. Related identifiers must be globally unique.

Identifier Type
The type of the related identifier
If a related identifier is used, identifier type is mandatory.
Relation Type
Description of the relationship of the resource being registered (A) and the related resource (B).
If a related identifier is used, relation type is mandatory. For a detailed explanation of the relationship selections, consult the full online help.
related Metadata Scheme
The name of the scheme; Use only with this relation pair: (HasMetadata/ IsMetadataFor)
Scheme URI
The URI (Location) of the scheme. Use only with this relation pair: (HasMetadata/ IsMetadataFor)
Scheme Type
The type of the scheme. Use only with this relation pair: (HasMetadata/ IsMetadataFor). Examples: XSD, DDT, Turtle

Rights

Any rights information regarding this resource.

Rights
Provide a rights management statement for the resource or reference a service providing such information. Include embargo information if applicable.
Use the complete title of a license and include version information if applicable. Example: Creative Commons Attribution 3.0 Germany License
URI
The URI (location) of the license.
Example: http://creativecommons.org/licenses/by/3.0/de/deed.en

Geolocation

Spatial region or named place where the data was gathered or about which the data is focused.

Repeat this property to indicate several different locations.

Point
A point contains a single latitude-longitude pair, separated by whitespace.
Use WGS 84 (World Geodetic System) coordinates. Use only decimal numbers for coordinates. Longitudes are -180 to 180 (0 is Greenwich, negative numbers are west, positive numbers are east), Latitudes are -90 to 90 (0 is the equator; negative numbers are south, positive numbers north). Example: 31.233 -67.302
Box
A box contains two white space separated latitude-longitude pairs, with each pair separated by whitespace. The first pair is the lower corner, the second is the upper corner.
Use WGS 84 (World Geodetic System) coordinates. Use only decimal numbers for coordinates. Longitudes are -180 to 180 (0 is Greenwich, negative numbers are west, positive numbers are east), Latitudes are -90 to 90 (0 is the equator; negative numbers are south, positive numbers north). Example: 41.090 -71.032 42.893 -68.211
Place
Free text. May be associated with the Point or Box, or may be used alone to describe a geographic location.

Size

Unstructured size information about the resource.

Examples: "15 pages", "6 MB"
